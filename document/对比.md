## 2. 核心追踪算法对比

### 2.1 主循环结构

```
RT.XD:                独立sbr:              H2hRT:
                      
for each ray:         for each ray:         for each ray:
  SbrFindPath(ray)      stack.push(root)      while alive:
                          while stack:          求交
  求交                    pop state            Rx检测
  Rx检测                  求交                  MonteCarlo R/T
  if hit:                Rx检测                 更新状态
    reflect → 递归        if hit:               continue
    pop_back               push(反射)
    transmit → 递归        push(透射)
    pop_back               push(绕射)
```

### 2.2 R/T 分裂策略 — 最核心差异

| | RT.XD | 独立sbr | H2hRT |
|---|-------|---------|-------|
| **策略** | 确定性双分支 | 确定性双分支 | **Monte Carlo 概率选择** |
| **机制** | 递归: 先R→递归→pop→再T→递归→pop | 栈DFS: 同时push R和T | 随机数r vs Fresnel反射率: r<refPwr→反射, r≥refPwr→透射 |
| **路径数/射线** | 2^depth (最大) | 2^depth (最大) | **1** |
| **完备性** | 高 (穷举所有R/T组合) | 高 (穷举所有R/T组合) | 中 (统计采样) |
| **效率** | O(rays × 2^depth) | O(rays × 2^depth) | **O(rays × depth)** |
| **确定性** | ✅ 可复现 | ✅ 可复现 | ❌ 随机 (需固定种子) |

**优劣分析**:
- **RT.XD / 独立sbr 的优势**: 路径完备, 不会遗漏低概率的透射路径, 适合精细信道仿真(FineChannel)
- **H2hRT 的优势**: 效率极高, 无分支爆炸, 适合大规模覆盖仿真(Coverage)
- **RT.XD 的独有特点**: 递归顺序 R 先于 T, power 不随分支衰减 (新射线继承原功率再各自乘系数)
- **独立sbr 的独有特点**: 栈顺序 R 和 T 平等push, 功率随 Fresnel 系数衰减

### 2.3 绕射处理

| | RT.XD | 独立sbr | H2hRT |
|---|-------|---------|-------|
| **策略** | 独立绕射模块 (SolveOneTimeDiffractionPathByEquation) | Keller锥耦合在追踪循环内 | Keller锥耦合在追踪循环内 |
| **触发方式** | 单独求解绕射方程 | 边命中检测 (点到边距离<eps) | 边命中检测 (barycentric) |
| **锥采样数** | 固定4方向 | 可配置 (diffraction_rays_per_event) | 固定4方向 |
| **UTD系数** | ✅ 完整 (Holm+, Luebbers, Fresnel积分) | ❌ 简化为标量0.08 | ❌ 简化的UTD标量 (1/(s1·s2·(s1+s2))) |
| **混合路径** | ❌ 绕射路径独立, 不与R/T混合 | ✅ R-D-T 混合 | ✅ R-D-T 混合 |
| **可见性验证** | ✅ | ❌ (Phase 4预留) | ✅ CheckSegment + RxGrid |

**优劣分析**:
- **RT.XD 的优势**: UTD绕射系数计算最完整 (Holm+启发式公式 + Fresnel函数数值积分), 论文级精度
- **RT.XD 的劣势**: 绕射与反射/透射分离, 不支持 R-D 混合路径 (如墙角绕射后反射)
- **独立sbr 的优势**: 绕射分支重新入栈可继续R/T交互, 支持任意顺序的混合路径
- **H2hRT 的优势**: 绕射与Rx检测一体化 (绕射方向直接查RxGrid), 效率高

## 3. 加速结构对比

| | RT.XD | 独立sbr | H2hRT |
|---|-------|---------|-------|
| **CPU后端** | CGAL AABB Tree | 自研 SAH-BVH (桶排序, 16桶) | 自研 FaceBVH |
| **GPU后端** | 无 | 无 (接口预留) | OptiX 7.7 Megakernel + Wavefront |
| **构建算法** | CGAL内置 | SAH桶分割 (3轴最优) | SAH逐点分割 |
| **遍历算法** | CGAL内置 | 距离优先 + 原子tMin早期终止 | 标准递归 |
| **射线-三角** | CGAL内置 | Möller-Trumbore | Möller-Trumbore (CPU) / OptiX内置 (GPU) |
| **构建速度** | 未测量 | 32ms / 20k面 | 未测量 |
| **遍历速度** | 未测量 | ~500μs/ray (20k面) | GPU: 批量百万级/帧 |

**优劣分析**:
- **独立sbr 的优势**: BVH算法完全自主可控, 零外部依赖, 桶排序SAH比逐点SAH更快, 距离优先遍历+原子tMin比标准递归更高效
- **H2hRT 的优势**: GPU OptiX后端, 批量射线查询性能远超CPU, 适合百万级射线
- **RT.XD 的优势**: CGAL成熟稳定, 但引入重型依赖

## 4. Rx检测对比

| | RT.XD | 独立sbr | H2hRT |
|---|-------|---------|-------|
| **方法** | 圆柱法 (射线为轴, 半径为Rx球) | 点到线段距离 < 球半径 | 3D均匀网格 (RxHashGrid) + 线段采样 |
| **复杂度** | O(N_rx) 每射线段 | O(N_rx) 每射线段 | O(1) 网格查询 + O(K) 候选验证 |
| **动态半径** | ❌ 固定 | ✅ Seidel-Rappaport模型 | ✅ Seidel-Rappaport模型 |
| **去重** | 无 | 后处理签名去重 | 线程级 generation counter 去重 |

**优劣分析**:
- **H2hRT 的优势**: RxHashGrid空间索引在大规模Rx (>100)时显著优于逐Rx线性扫描
- **独立sbr 的优势**: 动态接收球半径更物理 (射线管扩散模型), 减少假阳性/漏检
- **RT.XD 的优势**: 圆柱法实现简单, 但无空间索引, 大规模Rx时性能差

## 5. 材质与电磁计算对比

| | RT.XD | 独立sbr | H2hRT |
|---|-------|---------|-------|
| **材质模型** | 复介电常数 (ε_r, σ) → Fresnel | 折射率 n=√ε_r → Snell方向 + 法向Fresnel上界 | 复介电常数 → 完整Fresnel + 缓存 |
| **频率依赖** | ✅ 逐频点查询 | ✅ MaterialDatabase频率插值 | ✅ ITU-R P.2040外推 |
| **TIR处理** | ✅ 透射函数返回0 | ✅ SnellRefractV2自动TIR | ✅ sinT2>=1预检 + TIR→反射 |
| **透射功率** | 完整Fresnel透射系数 | 法向入射上界 T=1-R | 完整Fresnel透射系数 (1-refPwr) |
| **反射功率** | 完整Fresnel反射系数 | 法向入射上界 R=\|(n1-n2)/(n1+n2)\|² | 完整Fresnel反射系数 (refPwr缓存) |
| **绕射功率** | UTD Holm+启发式系数 | 标量0.08 | 简化UTD: 1/(s1·s2·(s1+s2)) |
| **几何/EM分离** | ❌ 一体化 (路径直接带dBm) | ✅ 纯几何输出 (GeometricPath→EM链路) | ✅ 几何寻径→EM求解分离 |

**优劣分析**:
- **RT.XD 的优势**: EM计算最完整 (完整Fresnel + UTD Holm+ + Fresnel函数数值积分), 论文级精度, 单次运行即得功率结果
- **RT.XD 的劣势**: 几何和EM耦合, 无法替换EM模型, 无法独立验证几何正确性
- **独立sbr 的优势**: 几何/EM完全解耦, GeometricPath 可输入任意EM求解器, 材质仅用于方向计算
- **H2hRT 的优势**: EM计算完整且模块化 (ApplyReflectionInteraction等独立模块), Fresnel带线程缓存
- **独立sbr 的劣势**: 功率仅为上界估计(法向入射), 不适用于精确链路预算

## 9. 架构建议

### 9.1 从RT.XD 可借鉴的特性

1. **UTD绕射系数**: 将 `CalculateDiffractionWaveLossHolmPlus` 移植到EM层 (非几何层)
2. **完整Fresnel**: 将逐角度Fresnel反射/透射系数用于功率估计
3. **递归追踪的简洁性**: 递归比栈DFS更易理解和维护, 适合深度较小的场景

### 9.2 从H2hRT 可借鉴的特性

1. **Monte Carlo R/T**: 作为Coverage模式的默认策略, 大幅提升效率
2. **RxHashGrid**: 大规模Rx (>100点) 时替换逐Rx检测
3. **GPU后端接口**: OptiX批量查询接口设计
4. **Fresnel缓存**: 线程局部缓存避免重复计算

### 9.3 独立sbr 的独特优势

1. **完全独立**: 零外部依赖, 纯C++17, 可编译为单文件库
2. **SAH-BVH**: 桶排序SAH比逐点SAH构建更快
3. **接口抽象**: ISceneAccelerator 支持任意后端替换
4. **JSON驱动**: 完整的JSON配置+输出体系, 适合自动化仿真管线
5. **Python可视化**: 配套pyvista可视化脚本
